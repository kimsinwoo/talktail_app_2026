## 1. 홈 화면 (Home Screen)

### 1.1 화면 목적
- 사용자가 앱을 열었을 때 즉시 오늘 해야 할 일과 반려동물의 상태를 파악
- 액션 중심의 대시보드로 행동 유도
- 여러 반려동물 간 전환 지원

### 1.2 주요 기능

#### 1.2.1 반려동물 프로필 영역
- **위치**: 화면 최상단
- **구성 요소**:
  - 반려동물 아바타 (이니셜 또는 사진)
  - 반려동물 이름
  - 상태 요약 (정성적 표현)
    - 예: "오늘 상태 체크 안 했어요"
    - 예: "평소와 비슷한 하루였어요"
- **인터랙션**:
  - 좌우 슬라이드로 여러 반려동물 전환
  - 좌우 화살표 버튼으로 수동 전환
  - 하단 페이지 인디케이터 (점 표시)

#### 1.2.2 코어 카드 ① - 데일리 건강 체크
- **위치**: 프로필 영역 하단
- **상태별 UI**:
  - **미완료 상태**:
    - 메인 텍스트: "오늘 ○○의 상태 체크가 아직 안 되었어요"
    - 서브 텍스트: "하루 한 번 기록이 차이를 만듭니다"
    - CTA 버튼: "오늘 상태 체크하기"
  - **완료 상태**:
    - 메인 텍스트: "오늘의 상태 체크 완료"
    - 서브 텍스트: "오늘의 식사, 산책, 컨디션 기록을 확인하세요"
    - CTA 버튼: "오늘 기록 보기"
- **동작**: 버튼 클릭 시 `DailyHealthCheckScreen` 또는 `RecentStatusTrendScreen`으로 이동

#### 1.2.3 코어 카드 ② - 일기
- **위치**: 데일리 건강 체크 카드 하단
- **상태별 UI**:
  - **미작성 상태**:
    - 메인 텍스트: "○○의 하루를 기록해볼까요?"
    - CTA 버튼: "일기 쓰기"
  - **작성 완료**:
    - 메인 텍스트: "오늘은 이랬어요"
    - 서브 텍스트: 일기 한 줄 미리보기
    - CTA 버튼: "기록 보기"
- **동작**: 버튼 클릭 시 `DiaryWriteScreen` 또는 `DiaryDetailScreen`으로 이동

#### 1.2.4 코어 카드 ③ - 최근 상태 흐름 요약
- **위치**: 일기 카드 하단
- **구성**:
  - 최근 건강 체크 기록 기반 정성적 요약
  - 예: "최근 3일간 식사량이 평소보다 적은 날이 있어요"
  - 예: "산책량이 줄어든 날이 자주 보여요"
- **동작**: 카드 클릭 시 `RecentStatusTrendScreen`으로 이동

#### 1.2.5 보조 카드 - 날씨 정보
- **위치**: 최근 상태 흐름 카드 하단
- **기본 상태**:
  - 날씨 요약 (예: "오늘은 산책하기 좋은 날씨예요")
  - 확장/축소 버튼
- **확장 상태**:
  - 온도, 습도, 풍속, 미세먼지 수치
  - 상세 날씨 정보

#### 1.2.6 서비스 아이콘 그리드
- **위치**: 날씨 카드 하단
- **레이아웃**: 3열 그리드
- **아이콘 목록**:
  1. 웨어러블 모니터링
  2. 피부 진단
  3. 근처 병원 찾기
  4. 이미지 생성
  5. 건강 리포트
  6. 건강 질문 도우미
- **동작**: 각 아이콘 클릭 시 해당 화면으로 이동

### 1.3 데이터 구조
```typescript
interface HomeScreenData {
  currentPet: Pet;
  dailyCheckStatus: {
    completed: boolean;
    completedAt?: Date;
  };
  diaryStatus: {
    hasToday: boolean;
    preview?: string;
  };
  recentTrendSummary: {
    insights: string[];
  };
  weather: {
    summary: string;
    temperature?: number;
    humidity?: number;
    windSpeed?: number;
    fineDust?: number;
  };
}
```

### 1.4 사용자 플로우
1. 앱 실행 → 홈 화면 진입
2. 반려동물 프로필 확인
3. 오늘의 상태 체크 미완료 확인 → "오늘 상태 체크하기" 클릭
4. 데일리 건강 체크 화면으로 이동
5. 체크 완료 후 홈 화면 복귀
6. 일기 작성 또는 다른 서비스 이용

### 1.5 UI/UX 원칙
- **액션 유도**: 정보 나열보다는 행동 유도
- **정성적 표현**: 수치/점수보다는 정성적 메시지
- **빠른 파악**: 3초 이내 오늘 해야 할 일 파악 가능
- **일관된 간격**: 전문적이고 세련된 UI를 위한 일관된 간격 시스템

## 2. 모니터링 화면 (Monitoring Screen)

### 2.1 화면 목적
- 웨어러블 디바이스를 통한 실시간 생체 신호 측정 및 모니터링
- BLE 1:1 연결 또는 허브(Hub) 기반 다중 디바이스 관리

### 2.2 주요 기능

#### 2.2.1 연결 모드 선택
- **BLE 1:1 연결 모드**:
  - 스마트폰과 웨어러블 디바이스 직접 연결
  - 단일 반려동물 모니터링
- **허브(Hub) 모드**:
  - 허브를 통한 다중 디바이스 관리
  - 여러 반려동물 동시 모니터링
  - Socket.IO 기반 실시간 통신

#### 2.2.2 실시간 데이터 표시
- **심박수 (Heart Rate)**:
  - 현재 심박수 수치
  - 실시간 그래프
  - 정상 범위 표시
- **산소포화도 (SpO2)**:
  - 현재 SpO2 수치
  - 정상 범위 표시
- **체온 (Temperature)**:
  - 현재 체온
  - 체온 추이 그래프
- **스트레스 지수 (Stress Index)**:
  - 현재 스트레스 수치
  - 수치 해석
- **칼로리 소모량**:
  - 일일 칼로리 소모량
  - 활동량 기반 계산

#### 2.2.3 측정 기능
- **기본 측정**:
  - 심박수, 산소포화도, 스트레스 지수 측정
  - 측정 시작/중지 버튼
  - 측정 시간 표시
- **활동 측정**:
  - 활동량 기반 칼로리 계산
  - 측정 결과 저장

#### 2.2.4 디바이스 상태
- **연결 상태**: 연결됨/연결 안 됨
- **배터리 상태**: 배터리 잔량 표시
- **신호 강도**: RSSI 표시

### 2.3 데이터 구조
```typescript
interface TelemetryData {
  deviceMac: string;
  timestamp: number;
  heartRate: number;
  spO2: number;
  temperature: number;
  stressIndex: number;
  battery: number;
}

interface MeasurementResult {
  type: 'basic' | 'activity';
  durationSec: number;
  sampleCount: number;
  avgHr: number;
  avgSpO2: number;
  stressIndex: number;
  calories?: number;
  score?: number;
}
```

### 2.4 사용자 플로우
1. 모니터링 탭 진입
2. 디바이스 연결 (BLE 또는 허브)
3. 측정 시작 버튼 클릭
4. 실시간 데이터 확인
5. 측정 중지 버튼 클릭
6. 측정 결과 확인 및 저장
7. 건강 리포트에 반영

### 2.5 UI/UX 원칙
- **실시간 피드백**: 데이터 수신 즉시 화면 업데이트
- **시각적 표현**: 그래프와 아이콘으로 직관적 표시
- **에러 처리**: 연결 실패 시 명확한 안내 메시지

## 3. 데일리 건강 체크 (Daily Health Check)

### 3.1 화면 목적
- 매일 간단하게 반려동물의 건강 상태를 기록
- 이상 징후의 조기 발견을 위한 일상 기록

### 3.2 주요 기능

#### 3.2.1 반려동물 정보 카드
- **위치**: 화면 상단
- **구성**:
  - 반려동물 아바타 (이니셜)
  - 반려동물 이름
  - 품종/종류
  - 오늘 날짜 (요일 포함)

#### 3.2.2 체크리스트 항목
1. **식사 상태**
   - 질문: "오늘 식사는 어땠나요?"
   - 선택지:
     - 잘 먹었어요
     - 평소보다 적었어요
     - 거의 안 먹었어요

2. **배변 상태**
   - 질문: "배변 상태는 어땠나요?"
   - 선택지:
     - 평소와 같아요
     - 조금 달랐어요
     - 많이 달랐어요

3. **활동량**
   - 질문: "오늘 활동량은 어땠나요?"
   - 선택지:
     - 평소와 비슷해요
     - 조금 적었어요
     - 많이 적었어요

4. **전반적인 컨디션**
   - 질문: "오늘 컨디션은 어땠나요?"
   - 선택지:
     - 좋아 보여요
     - 평소와 비슷해요
     - 안 좋아 보여요

5. **특이사항** (선택)
   - 질문: "특별히 신경 쓰인 점이 있었나요?"
   - 선택지:
     - 없음
     - 조금 있었어요
     - 있었어요
   - **조건부 입력**: "있었어요" 선택 시 텍스트 입력 필드 표시

#### 3.2.3 저장 기능
- **저장 버튼**: 모든 필수 항목 선택 시 활성화
- **저장 후 동작**: 홈 화면으로 복귀 및 성공 메시지 표시

### 3.3 데이터 구조
```typescript
interface DailyHealthCheck {
  petCode: string;
  date: string; // YYYY-MM-DD
  meal: 'good' | 'less' | 'little';
  poop: 'normal' | 'slightly' | 'different';
  activity: 'similar' | 'less' | 'much_less';
  condition: 'good' | 'normal' | 'bad';
  specialNote?: string;
}
```

### 3.4 사용자 플로우
1. 홈 화면에서 "오늘 상태 체크하기" 클릭
2. 데일리 건강 체크 화면 진입
3. 각 항목별 선택지 선택 (라디오 버튼 형식)
4. 특이사항이 있으면 텍스트 입력
5. 저장 버튼 클릭
6. 홈 화면으로 복귀

### 3.5 UI/UX 원칙
- **직관적 표현**: 의료 용어 대신 보호자가 이해하기 쉬운 표현
- **빠른 입력**: 3초 이내 입력 완료 가능
- **정성적 평가**: 수치화보다는 정성적 판단 중심
- **선택적 입력**: 특이사항은 선택 사항

## 4. 최근 상태 흐름 (Recent Status Trend)

### 4.1 화면 목적
- 최근 건강 체크 기록을 분석하여 패턴과 추세 파악
- 이상 징후 조기 발견을 위한 데이터 시각화

### 4.2 주요 기능

#### 4.2.1 반려동물 정보 카드
- **위치**: 화면 상단
- **구성**:
  - 반려동물 아바타
  - 반려동물 이름
  - 최근 기록 기간 표시 (예: "최근 7일간의 기록")

#### 4.2.2 패턴 분석 섹션
- **위치**: 반려동물 카드 하단
- **구성**:
  - 최근 기록 기반 정성적 인사이트
  - 예: "최근 7일간 식사량이 평소보다 적은 날이 3일 있어요"
  - 예: "산책량이 줄어든 날이 자주 보여요"
  - 예: "배변 상태가 평소와 다른 날이 2일 있었어요"
- **표시 방식**: 불릿 포인트 리스트

#### 4.2.3 일별 기록 테이블
- **위치**: 패턴 분석 섹션 하단
- **구성**:
  - 테이블 헤더: 날짜, 식사, 배변, 활동, 컨디션
  - 각 행: 날짜별 건강 체크 기록
  - 상태 아이콘:
    - ✅ (좋음): 초록색 CheckCircle2
    - ➖ (보통): 회색 Minus
    - ❌ (나쁨): 빨간색 XCircle
  - 특이사항이 있는 날: 왼쪽 테두리 강조 표시

#### 4.2.4 특이사항 섹션
- **위치**: 일별 기록 테이블 하단
- **조건**: 특이사항이 기록된 날이 있을 때만 표시
- **구성**:
  - 날짜별 특이사항 목록
  - 날짜와 내용 표시

#### 4.2.5 안내 메시지
- **위치**: 화면 하단
- **내용**: "이 기록은 보호자의 관찰을 바탕으로 한 정보입니다. 이상 징후가 지속되면 수의사와 상담하시는 것을 권장합니다."

### 4.3 데이터 구조
```typescript
interface DailyCheckRecord {
  date: string; // YYYY-MM-DD
  meal: 'good' | 'less' | 'little';
  poop: 'normal' | 'slightly' | 'different';
  activity: 'similar' | 'less' | 'much_less';
  condition: 'good' | 'normal' | 'bad';
  specialNote?: string;
}

interface PatternInsight {
  type: 'meal' | 'activity' | 'poop' | 'condition';
  message: string;
  count: number;
}
```

### 4.4 사용자 플로우
1. 홈 화면에서 "최근 상태 흐름" 카드 클릭
2. 최근 상태 흐름 화면 진입
3. 패턴 분석 확인
4. 일별 기록 테이블 확인
5. 특이사항 확인 (있는 경우)
6. 필요 시 건강 질문 도우미 또는 병원 찾기로 이동

### 4.5 UI/UX 원칙
- **정성적 표현**: 수치보다는 정성적 메시지
- **시각적 구분**: 아이콘으로 상태를 직관적으로 표시
- **특이사항 강조**: 특이사항이 있는 날은 시각적으로 강조

## 5. 일기 관련 화면들

### 5.1 일기 목록 화면 (Diary Screen)

#### 5.1.1 화면 목적
- 반려동물의 일기 목록을 날짜순으로 표시
- 일기 작성 및 상세 보기 진입점

#### 5.1.2 주요 기능
- **일기 목록**: 날짜별 일기 카드 표시
- **일기 카드 구성**:
